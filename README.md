# VnNotes — консольная система заметок и мониторинга

VnNotes — консольное приложение для ведения служебных заметок в ИТ-инфраструктуре VPN-сервиса.

Приложение предназначено для запуска на устройствах ИТ-инфраструктуры: серверах приложений, серверах баз данных, шлюзах, балансировщиках, рабочих станциях и других узлах. Система позволяет авторизованному пользователю создавать служебные заметки, просматривать состояние устройства, сохранять статистику нагрузки и контролировать события безопасности.

Проект разработан на C# с использованием PostgreSQL для хранения пользователей, ролей, заметок, статистики нагрузки и журнала безопасности.

---

## Основные возможности

- авторизация пользователей;
- разграничение прав по ролям `admin`, `operator`, `user`;
- хранение пользователей в PostgreSQL;
- MD5-хеширование паролей;
- создание заметок;
- просмотр заметок;
- удаление заметок;
- создание пользователей администратором;
- разблокировка пользователей;
- регистрация узлов ИТ-инфраструктуры;
- просмотр текущей статистики CPU, RAM и HDD;
- сохранение статистики нагрузки в PostgreSQL;
- просмотр последней сохраненной статистики по узлам;
- ведение журнала безопасности;
- просмотр журнала безопасности;
- проверка обновлений через GitHub;
- загрузка обновления из GitHub Releases;
- применение обновления через отдельный модуль `VnUpdater`;
- создание Markdown-карты команд.

## Настройка PostgreSQL

Перед запуском приложения необходимо создать базу данных и отдельного пользователя для программы.

Создание базы данных:

```sql
CREATE DATABASE vn_notes_db;
```

Создание пользователя приложения:

```sql
CREATE ROLE vn_app WITH LOGIN PASSWORD 'vn_password';
```

Выдача права на подключение к базе:

```sql
GRANT CONNECT ON DATABASE vn_notes_db TO vn_app;
```

После этого нужно подключиться к базе `vn_notes_db` и выдать права на схему `public`:

```sql
GRANT USAGE, CREATE ON SCHEMA public TO vn_app;
```

## Строка подключения

Строка подключения находится в файле `App.config`:

```xml
<add key="ConnectionString" value="Host=localhost;Port=5432;Database=vn_notes_db;Username=vn_app;Password=vn_password" />
```

Текущая версия приложения также хранится в `App.config`:

```xml
<add key="ApplicationVersion" value="1.0.1" />
```

Ссылка на файл последней версии на GitHub:

```xml
<add key="UpdateInfoUrl" value="https://raw.githubusercontent.com/FiLLLLiN/VnNotes/main/version.json" />
```

## Принцип работы приложения

После запуска приложение выполняет следующие действия:

```text
1. Загружает настройки из App.config.
2. Проверяет наличие обновлений через version.json на GitHub.
3. Подключается к PostgreSQL.
4. Инициализирует таблицы базы данных.
5. Запрашивает логин и пароль пользователя.
6. После успешной авторизации открывает интерактивный режим команд.
```

После входа пользователь работает с программой через приглашение:

```text
vn>
```

## Обновление приложения

Обновление реализовано через GitHub Releases.

В корне репозитория находится файл:

```text
version.json
```

Пример содержимого:

```json
{
  "version": "1.0.1",
  "downloadUrl": "https://github.com/FiLLLLiN/VnNotes/releases/download/v1.0.1/VnNotes-v1.0.1.zip",
  "releaseNotes": "Исправлена справка команд, улучшена работа интерактивного режима и обновлений."
}
```

Программа сравнивает локальную версию из `App.config` с версией из `version.json`.

Если версия на GitHub больше локальной версии, приложение сообщает о наличии обновления.

Для загрузки обновления используется команда:

```text
update-download
```

Для применения обновления используется команда:

```text
update-apply
```

После запуска обновления основное приложение закрывается, а отдельный модуль `VnUpdater` распаковывает архив и заменяет файлы приложения.

## Подготовка новой версии

Для выпуска новой версии необходимо:

```text
1. Изменить код приложения.
2. Обновить версию в App.config.
3. Собрать проект в режиме Release.
4. Создать zip-архив готовой сборки.
5. Загрузить архив в GitHub Releases.
6. Обновить version.json.
7. Зафиксировать изменения в Git.
```

Архив должен называться по шаблону:

```text
VnNotes-v1.0.1.zip
```

Внутри архива должны лежать файлы программы без лишней вложенной папки `Release`:

```text
VnNotes.exe
VnNotes.exe.config
VnUpdater.exe
Npgsql.dll
database/
```
